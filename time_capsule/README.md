# Капсула Времени ⏳

Проект для Яндекс Лицея по направлению WEB. Веб-приложение для создания и хранения капсул времени с возможностью открытия в указанную дату.

## 🚀 Особенности проекта

### Уникальность:
- **Отложенное открытие**: Контент капсулы скрыт до наступления указанной даты
- **Категоризация**: 4 типа капсул (личное, воспоминание, цель, послание)
- **REST API**: Полноценный API для интеграции с другими сервисами
- **Telegram Bot**: Уведомления о готовых к открытию капсулах
- **Статистика**: Отслеживание созданных и открытых капсул

### Технологии:
- **Flask** - веб-фреймворк (WEB 1)
- **Flask-WTF** - обработка форм (WEB 2)
- **Flask-SQLAlchemy** - работа с базой данных (WEB 3, WEB 4)
- **Flask-RESTful** - создание REST API (WEB 5, WEB 6)
- **python-telegram-bot** - Telegram бот
- **SQLite** - база данных
- **HTML/CSS** - адаптивный дизайн

## 🛠️ Установка и запуск

### 1. Установка зависимостей

```bash
cd time_capsule
pip install -r requirements.txt
```

### 2. Инициализация базы данных

```bash
flask init-db
```

Или просто запустите приложение - база создастся автоматически.

### 3. Запуск приложения

```bash
python app.py
```

Приложение будет доступно по адресу: http://localhost:5000

### 4. Запуск Telegram бота (опционально)

1. Получите токен бота от @BotFather в Telegram
2. Откройте `bot.py` и замените `YOUR_BOT_TOKEN_HERE` на ваш токен
3. Запустите бота:

```bash
python bot.py
```

## 📡 REST API

### Получить все капсулы
```
GET /api/capsules
```

### Получить одну капсулу
```
GET /api/capsules/<id>
```

### Создать капсулу
```
POST /api/capsules
Content-Type: application/json

{
    "title": "Моя капсула",
    "content": "Содержимое капсулы",
    "author": "Имя автора",
    "open_at": "2025-12-31T23:59:00",
    "category": "personal"
}
```

### Обновить капсулу
```
PUT /api/capsules/<id>
Content-Type: application/json

{
    "title": "Новое название",
    "is_opened": true
}
```

### Удалить капсулу
```
DELETE /api/capsules/<id>
```

### Получить статистику
```
GET /api/stats
```

## 🔧 Команды CLI

### Проверка готовых капсул
```bash
flask check-capsules
```

Выведет список капсул, которые можно открыть.

## 📝 Примеры использования

### Через веб-интерфейс:
1. Откройте http://localhost:5000
2. Заполните форму создания капсулы
3. Укажите дату открытия в будущем
4. Сохраните ID капсулы
5. Вернитесь после указанной даты для открытия

### Через API:
```bash
# Создание капсулы
curl -X POST http://localhost:5000/api/capsules \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Послание себе",
    "content": "Не забудь о своих мечтах!",
    "author": "Я",
    "open_at": "2025-01-01T00:00:00",
    "category": "goal"
  }'

# Проверка статистики
curl http://localhost:5000/api/stats
```

## 📊 Соответствие темам курса

| Тема | Реализация |
|------|------------|
| WEB 1: Введение во flask, обработка форм | ✅ Формы создания и открытия капсул |
| WEB 2: Шаблоны, flask-wtf | ✅ Jinja2 шаблоны, валидация форм |
| WEB 3-4: Flask-sqlalchemy | ✅ Модели Capsule и Tag, связи many-to-many |
| WEB 5: REST-API | ✅ Полный CRUD через API |
| WEB 6: Flask-restful | ✅ Resource классы для API |
| WEB 7: Развёртывание | ✅ Готовность к деплою (gunicorn в requirements) |
| HTTP-API задание | ✅ JSON API, командная строка |
